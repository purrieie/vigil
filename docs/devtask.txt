Task 2.6: Groq Llama 3 structured-JSON prompt + Pydantic validation + one-retry logic
Task 2.7: Graceful degradation for missing modality (audio-only / image-only / text-only)
Task 2.8: Confidence-tiered verdict mapping (Critical/Caution/Clear per thresholds)
Task 2.9: Static fallback JSON payload switch (bypass live models if triggered)
Task 2.10 (Frontend): SandboxIntake page — fake WhatsApp thread UI, attachment picker
Task 2.11 (Frontend): Step-by-step "analyzing" loading state
Task 2.12 (Frontend): VerdictCard component (Critical/Caution/Clear variants)

Dependencies: Epic 1 complete

Acceptance Criteria:

Given audio-only input, system returns a verdict without error
Given a forced vision-model failure, system still returns a verdict with degraded_extraction: true logged
Given the demo fallback flag is on, response arrives from static payload, not live models

Dependencies: Epic 1; Epic 2 (needs extraction output format finalized before writer can consume it)

Graph interaction (zoom/pan/click) responds at >30fps with full seed dataset loaded
